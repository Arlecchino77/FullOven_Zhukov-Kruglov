# Мини-ТЗ: TaskTracker

## 1. Название
FullOven (Сайт сети ресторанов)

## 2. Цель
Программа помогает ознакомиться с нашим меню и узнать близжайшую точку рядом с домом.

## 3. Пользователь
Обычный пользователь ПК (клиент).

## 4. Функциональные требования (FR)
- FR1: О ресторане (FullOven)
- FR2: Наши места на карте
- FR3: Открыть меню
- FR4: Закрыть меню 
- FR5: Заброниовать стол при помощи горячей линии
- FR6: Наши контакты (номер телефона/почта)
- FR7: Выход

## 5. Нефункциональные требования (NFR)
- Приложение запускается без ошибок
- Понятные сообщения об ошибках (например: “Пустое название нельзя”)
- Данные будут храниться локально (позже: JSON-файл)

## 6. Модель данных (черновик)
**Dish(Блюдо)**
- Id (число)
- Name (строка, не пустая)
- Price (число)
- Category (строка: Супы/ Горячее/ Десерты/ Напитки)

**Booking(Бронирование)**
- Id(число)
- ClientName(строка, но не путая)
- Phone(строка,но не пустая)
- Date(дата)
- Time(время)
- GuestsCount(число > 0)
- Status(New/ Confirmed/ Cancelled)

## 7. Критерии приемки (как проверить)
- Можно открыть сайт и перейти на основные страницы(Главная, Меню, Контакты)
- Меню загружается из данных и отображает список блюд
- Можно заполнить форму бронирования стола, и заявка сохранится
- Администратор видит список всех бронирований
- Пользователь получает понятные сообщения об ошибках

## 8. План модулей (черновик)
- Core (Модель: Dish, Booking, Логика: валидация, правила работы с данными)
- App (консольное меню)
- Storage (Сохранение и загрузка блюд и бронирований)
